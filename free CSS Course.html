CSS BASICS

Learn CSS syntax, selectors, and how to style HTML

Beginner

Module 1: CSS Syntax & Selectors

CSS (Cascading Style Sheets) controls the presentation of HTML elements. It uses rules: a selector points to an HTML element, and a declaration block sets styles.

Selectors can target element types, classes (.class), IDs (#id), or attributes. You can include CSS inline, inside <style> tags, or in external files.

Try It Yourself:

HTML+CSS
Live Preview

Key Points:

  • CSS rules consist of a selector and declaration block.
  • Use classes (.class) for reusable styles.
  • IDs (#id) are unique and have higher specificity.
  • Comments: /* comment */.

CSS Fundamentals

CSS cascades: later rules override earlier ones unless specificity differs. Understanding selectors and specificity is essential for maintainable styles.

Specificity

IDs have highest specificity, then classes/attributes, then element selectors. Use low specificity for reusability.

Inheritance

Some properties (color, font) inherit from parent elements, saving code.

Best Practice

Use class selectors over IDs for styling to maintain flexibility.

Real-World Application

Every website uses CSS selectors. Frameworks like Bootstrap rely heavily on class selectors for reusable components.

Master color values and background properties

Beginner

Module 2: Colors, Backgrounds & Gradients

CSS offers many ways to define colors: named, hex, RGB, RGBA, HSL, HSLA. Background properties include background-color, background-image, background-repeat, background-position, and background-size.

Try It Yourself:

Live Preview

Key Points:

  • color for text, background-color for background.
  • Colors: named, hex, rgb(), rgba(), hsl(), hsla().
  • Gradients: linear-gradient(), radial-gradient().
  • Background images can be positioned and sized.

Color Theory & Backgrounds

Accessible contrast is crucial. Use RGBA/HSLA for transparency. Gradients add depth without extra images. Optimize background images for performance.

Accessibility

Performance

Use CSS gradients instead of large images where possible.

Modern Techniques

CSS variables can store color values for easy theming.

Real‑World

Modern UI often uses subtle gradients and semi-transparent overlays for cards and modals.

BOX MODEL

Understand margins, borders, padding, and content

Beginner

Module 3: The CSS Box Model

Every HTML element is a rectangular box. The box model consists of: content, padding, border, and margin. Understanding how they interact is key to layout.

Try It Yourself:

HTML+CSS
Live Preview

Key Points:

  • margin outer space, border wraps padding, padding inner space.
  • box-sizing: border-box makes width include padding and border.
  • Vertical margins collapse between adjacent elements.

Understanding Layout

The box model is the foundation of CSS layout. Use box-sizing: border-box to avoid unexpected dimensions. Margins collapse can be controlled with padding or border on the parent.

Margin Collapse

Vertical margins of adjacent blocks combine into the larger margin.

Inline vs Block

Inline elements don’t respect vertical margins/padding fully.

Reset

Many resets apply box-sizing: border-box globally.

Real‑World

Layout frameworks like Bootstrap use border-box to simplify sizing.

FLEXBOX

Create flexible, responsive layouts with Flexbox

Intermediate

Module 5: Flexible Box Layout

Flexbox is a one‑dimensional layout model that distributes space along a row or column. It makes it easy to align items, distribute space, and create responsive designs without floats or positioning.

Try It Yourself:

HTML+CSS
Live Preview

Key Points:

  • display: flex creates a flex container.
  • flex-direction sets row or column.
  • justify-content aligns items on main axis.
  • align-items aligns on cross axis.
  • flex-wrap allows items to wrap.

Modern Layouts with Flexbox

Flexbox simplifies centering, equal‑height columns, and responsive navigation. It's ideal for components like navbars, cards, and forms.

Centering

Use justify-content: center; align-items: center; to center items perfectly.

Ordering

order property changes visual order without changing HTML.

Gap

gap property adds consistent spacing between flex items.

Real‑World

Flexbox powers many UI patterns: navigation bars, media objects, card grids, and responsive layouts.

POSITIONING & DISPLAY

Control element placement with positioning properties

Intermediate

Module 7: Positioning & Display Values

The position property allows you to take elements out of normal flow. Values: static, relative, absolute, fixed, sticky. display controls whether an element is block, inline, flex, grid, etc.

Try It Yourself:

HTML+CSS
Live Preview

Key Points:

  • position: relative offsets element from normal position.
  • position: absolute positions relative to nearest positioned ancestor.
  • position: fixed positions relative to viewport.
  • position: sticky toggles between relative and fixed based on scroll.

Advanced Positioning

Absolute positioning is powerful for overlays, tooltips, and modals. Sticky headers improve navigation. Always consider z‑index for stacking order.

Z‑Index

z-index controls stacking order. Only works on positioned elements.

Containing Blocks

Absolute elements are positioned relative to the nearest positioned ancestor (non‑static).

Display

display: none removes element from layout; visibility: hidden hides but preserves space.

Real‑World

Sticky navigation bars, fixed headers, modal popups, and tooltips use these positioning techniques.

RESPONSIVE DESIGN

Make websites adapt to any screen size

Advanced

Module 9: Media Queries & Responsive Techniques

Responsive design ensures websites work on all devices. Use media queries to apply styles based on screen width, height, orientation, etc. Combine with fluid layouts, flexible images, and relative units.

Try It Yourself:

HTML+CSS
Live Preview

Key Points:

  • @media (max-width: ...) targets screens smaller than a breakpoint.
  • Use relative units (%, vw, rem) for fluid layouts.
  • Responsive images: max-width: 100% ensures images scale.
  • Mobile‑first design: start with mobile styles, then add breakpoints for larger screens.

Designing for All Devices

Modern websites use a mobile‑first approach. Breakpoints are typically based on common device widths (576px, 768px, 992px, 1200px). Test on real devices or browser dev tools.

Fluid Grids

Use fr units, calc(), or flex-wrap for fluid columns.

Viewport Meta

<meta name="viewport"> is essential for responsive design.

Responsive Images

Use srcset and picture for different resolutions.

Real‑World

All major websites (Amazon, BBC, etc.) implement responsive design to serve mobile, tablet, and desktop users seamlessly.

Advanced CSS includes custom properties (CSS variables), complex selectors, functions like calc(), clamp(), min(), max(), and techniques like CSS Houdini, container queries, and logical properties.

  • CSS custom properties (--var) enable dynamic theming.
