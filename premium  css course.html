1.1 What is CSS?

8 min
Beginner

Chapter Explanation

CSS (Cascading Style Sheets) is the language that brings life to HTML documents. While HTML creates the structure and content of a webpage, CSS controls its visual presentation. Think of HTML as the skeleton and CSS as the skin, clothes, and makeup.

The "cascading" part refers to how styles are applied in layers, with more specific rules overriding general ones. This allows for systematic styling where you can define global styles and override them locally when needed.

CSS works by selecting HTML elements and applying style rules to them. These rules can control everything from colors and fonts to complex layouts and animations.

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of HTML documents. While HTML provides the structure, CSS controls how that structure looks: colors, layout, fonts, spacing, and animations.

How CSS Works

CSS selects HTML elements and applies style rules. These rules cascade, meaning multiple rules can apply to the same element, with specific rules overriding general ones.

CSS Example

    font-size: 2.5rem;
    text-align: center;

p {
    line-height: 1.6;
    color: #333;

1.2 Inline, Internal & External CSS

10 min
Beginner

Chapter Explanation

CSS can be applied to HTML documents in three different ways, each with its own use cases and best practices:

1. Inline CSS: Applied directly to individual HTML elements using the style attribute. Quick for testing but hard to maintain for large projects.

2. Internal CSS: Defined within a <style> tag in the HTML document's <head>. Useful for single-page applications or when styles are specific to one page.

3. External CSS: Stored in separate .css files and linked to HTML documents. This is the most maintainable approach for production websites as it promotes reusability and separation of concerns.

Best Practice: For production websites, always use external CSS files. This improves loading performance (browsers can cache CSS files) and makes your code more organized and maintainable.

Three ways to apply CSS to HTML:

1. Inline CSS

Applied directly to elements using style attribute.

Inline CSS 
<h1 style="color: blue; font-size: 2.5rem;">
    This is a blue heading
</h1>

2. Internal CSS

Inside <style> tag in HTML <head>.

<style>
    body {
        font-family: Arial;
        background: #f5f5f5;
    }
    .btn {
        background: #00C853;
        color: white;
    }
</style>

3. External CSS

Separate .css file linked via <link>.

<link rel="stylesheet" href="styles.css">

  • Directly on elements

3. What attribute is used for inline CSS?

  • class

Answer: style. The style attribute contains CSS rules directly on HTML elements.

1.3 CSS Syntax & Structure

12 min
Beginner

Chapter Explanation

CSS follows a specific syntax that consists of selectors and declaration blocks. Understanding this structure is fundamental to writing effective CSS.

Selector: Targets the HTML element(s) you want to style. This can be an element name, class, ID, or more complex pattern.

Declaration Block: Contains one or more declarations surrounded by curly braces {}.

Declaration: A property-value pair that defines a specific style. Each declaration ends with a semicolon.

Property: The aspect of the element you want to change (like color, font-size).

Value: The setting for the property (like red, 16px).

CSS is case-insensitive (except for certain values like URLs) but it's best practice to use consistent casing. Semicolons are required to separate declarations, and whitespace is generally ignored.

CSS rule = selector + declaration block with property-value pairs.

CSS Rule 
selector {
    property: value;
    another-property: another-value;
}

/* Example */
h1 {
    color: #FF6B35;
    font-size: 2.5rem;
    text-align: center;
}

Multiple Selectors

h1, h2 {
    font-family: 'Inter', sans-serif;
    color: #1a1f2e;
    margin-bottom: 1rem;
}

1.4 CSS Selectors

15 min
Beginner

Chapter Explanation

CSS selectors are patterns used to select and style HTML elements. They are one of the most powerful features of CSS, allowing precise targeting of elements.

Basic Selectors:

  • Element selector: Targets all instances of an HTML element (e.g., p, h1)
  • Class selector: Targets elements with a specific class (prefixed with .)
  • ID selector: Targets a single element with a specific ID (prefixed with #)
  • Attribute selector: Targets elements with specific attributes

Combinator Selectors: Combine multiple selectors to target specific relationships:

  • Descendant: div p (all p inside div)
  • Child: div > p (direct children p of div)
  • Adjacent sibling: h1 + p (p immediately after h1)
  • General sibling: h1 ~ p (all p after h1)

Pseudo-classes: Select elements in specific states (like :hover, :focus)

Pseudo-elements: Style specific parts of elements (like ::before, ::after)

Basic Selectors

/* Element */
p { color: #333; }

/* Class */
.button { padding: 12px 24px; }

/* ID */
#header { background: var(--dark); }

/* Attribute */
input[type="text"] { padding: 10px; }

/* Pseudo-class */
a:hover { color: var(--primary); }
tr:nth-child(odd) { background: #f9f9f9; }

/* Descendant */
article p { line-height: 1.8; }

/* Child */
nav > ul { list-style: none; }

/* Adjacent sibling */
h2 + p { margin-top: 0; }

  • ul elements anywhere inside nav
  • nav elements inside ul
Answer: ul elements that are direct children of nav. The > symbol selects direct children only.

3. Which pseudo-class would you use to style a link when the mouse is over it?

Answer: :hover. This pseudo-class applies when the user hovers over an element.

Well-organized and commented CSS is crucial for maintainability, especially in team environments or large projects.

CSS Comments: Use /* comment */ syntax. Comments are ignored by browsers but essential for developers to understand the code. They can be single-line or multi-line.

Organization Strategies:

  • Group related rules: Keep all typography styles together, all button styles together, etc.
  • Logical ordering: Start with reset/normalize, then base elements, then components, then utilities
  • Use sections: Divide your CSS into clear sections with comment headers
  • Naming conventions: Use consistent naming like BEM (Block, Element, Modifier)

BEM Methodology: A popular naming convention that creates clear relationships between CSS and HTML:

  • Block: Standalone component (e.g., .card)
  • Element: Part of a block (e.g., .card__title)
  • Modifier: Variation of a block or element (e.g., .card--featured)

Comments: /* This is a comment */

Organization

    line-height: 1.6;

/* ===== BUTTONS ===== */

    border-radius: 8px;
}

/* BEM Naming */
.card { }
.card__title { }
.card--featured { }

CSS provides various ways to specify colors, units, and values, each with specific use cases.

Color Formats:

  • Named colors: Basic colors like red, blue (limited options)
  • HEX: #RRGGBB or #RGB - most common for web
  • RGB/RGBA: rgb(red, green, blue) with optional alpha (transparency)
  • HSL/HSLA: Hue, Saturation, Lightness - more intuitive for color manipulation

CSS Units:

  • Absolute units: Fixed sizes (px, pt, cm) - best for print styles
  • Relative units: Scale relative to something else:
    • %: Percentage of parent element
    • em: Relative to parent font-size
    • rem: Relative to root (html) font-size
    • vw/vh: Viewport width/height percentage
  • Modern functions: min(), max(), clamp() for responsive values

Best Practice: Use relative units (rem, %) for most layouts to ensure scalability and accessibility.

Color Formats

/* Named */

/* HSL/HSLA */

color: hsla(145, 100%, 39%, 0.8);

2.1 Box Model

12 min
Intermediate

Chapter Explanation

The CSS Box Model is a fundamental concept that describes how every element in HTML is represented as a rectangular box. Understanding this model is crucial for proper layout and spacing.

The Box Model consists of four layers from inside out:

  1. Content: The actual content (text, images, etc.)
  2. Padding: Space between content and border (inside the element)
  3. Border: Line surrounding the padding (if any)
  4. Margin: Space outside the border (between elements)

Box-Sizing Property: Controls how element dimensions are calculated:

  • content-box (default): width/height = content only
  • border-box: width/height = content + padding + border

Pro Tip: Always set box-sizing: border-box globally using the universal selector *. This makes layout calculations much more predictable.

Every element is a rectangular box with: content, padding, border, margin.

Box Model 
.box {
    /* Content dimensions */
    width: 300px;
    height: 200px;
    
    /* Padding: space inside border */
    padding: 20px;
    
    /* Border: line around padding */
    border: 2px solid #00C853;
    
    /* Margin: space outside border */
    margin: 30px;
    
    /* Box-sizing */
    box-sizing: border-box; /* width includes padding+border */
    box-sizing: content-box; /* default */
}

Chapter Quiz

1. In the CSS Box Model, what comes between the content and the border?

  • Margin
  • Padding
  • Outline
  • Shadow
Answer: Padding. Padding is the space between the content and the border.

2. With box-sizing: border-box, what does the width property include?

  • Content only
  • Content + padding
  • Content + padding + border
  • Content + padding + border + margin
Answer: Content + padding + border. This makes layout calculations much easier.

3. Which part of the box model creates space between elements?

  • Padding
  • Border
  • Margin
  • Content
Answer: Margin. Margins create space outside the border, between elements.

2.2 Margin & Padding

10 min
Intermediate

Chapter Explanation

Margin and padding are essential spacing properties that control the space around and within elements.

Padding: Space inside an element, between its content and its border. Padding affects the element's background and clickable area.

Margin: Space outside an element, between its border and neighboring elements. Margins can collapse (overlap) vertically.

Shorthand Syntax: Both properties support shorthand notation:

  • One value: Applied to all four sides
  • Two values: First = top/bottom, Second = left/right
  • Three values: First = top, Second = left/right, Third = bottom
  • Four values: top, right, bottom, left (clockwise)

Margin Collapse: When two vertical margins meet, they collapse into a single margin equal to the larger of the two. Horizontal margins don't collapse.

Auto Margins: Used to center elements horizontally (margin: 0 auto).

Negative Margins: Can pull elements closer together or create overlapping effects.

Margin & Padding 
/* One value: all sides */
margin: 20px;
padding: 15px;

/* Two values: vertical | horizontal */
margin: 10px 20px;
padding: 15px 30px;

/* Four values: top | right | bottom | left */
margin: 10px 20px 15px 5px;
padding: 15px 10px 20px 5px;

/* Individual sides */
margin-top: 10px;
margin-right: 15px;
margin-bottom: 20px;
margin-left: 5px;

padding-top: 10px;
padding-right: 15px;
padding-bottom: 20px;
padding-left: 5px;

/* Auto center */
margin: 0 auto; /* center horizontally */

/* Negative margin */
margin-top: -10px;

1. What does "margin: 10px 20px;" mean?

  • Top/bottom: 10px, Left/right: 20px
  • Top: 10px, Right: 20px, Bottom: 10px, Left: 20px
  • All sides: 10px then 20px
  • Top/left: 10px, Bottom/right: 20px
Answer: Top/bottom: 10px, Left/right: 20px. Two values mean vertical | horizontal.

2. How do you horizontally center a block element?

  • margin: auto;
  • margin: 0 auto;
  • padding: 0 auto;
  • align: center;
Answer: margin: 0 auto;. This sets top/bottom margins to 0 and left/right to auto.

3. What happens to vertical margins of adjacent elements?

  • They add together
  • They collapse (the larger one wins)
  • They multiply
  • They create a gap equal to their sum
Answer: They collapse (the larger one wins). This is called margin collapse.

2.3 Borders & Outlines

12 min
Intermediate

Chapter Explanation

Borders and outlines add visual definition to elements, but they work differently and serve different purposes.

Borders: Part of the box model that sits between padding and margin. Borders:

  • Affect element dimensions (unless using box-sizing: border-box)
  • Can have different styles on each side
  • Participate in box model calculations
  • Can be rounded with border-radius

Border Properties:

  • border-width: Thickness of the border
  • border-style: solid, dashed, dotted, double, etc.
  • border-color: Color of the border
  • border-radius: Rounds the corners (can create circles with 50%)

Outlines: Drawn outside borders, don't affect layout or dimensions. Useful for:

  • Focus indicators for accessibility
  • Highlighting elements without affecting layout
  • Temporary visual feedback

Key Difference: Borders affect element size and layout; outlines don't. You can have both simultaneously.

Borders 
/* Shortcut */
border: 2px solid #00C853;

border-width: 2px;
border-style: solid;
border-color: #00C853;

/* Border sides */
border-top: 1px solid red;
border-right: 2px dashed blue;
border-bottom: 3px dotted green;
border-left: 4px double orange;

/* Border radius */
border-radius: 10px;
border-radius: 50%; /* circle */
border-radius: 10px 20px 30px 40px; /* TL TR BR BL */

/* Outline (outside border, doesn't affect layout) */
outline: 2px solid #FF6B35;
outline-offset: 5px;

2.4 Width & Height

10 min
Intermediate

Chapter Explanation

Controlling element dimensions is fundamental to layout design. CSS provides various ways to set width and height, each with specific behaviors.

Fixed Dimensions: Using absolute units like px. Predictable but not flexible for responsive design.

Relative Dimensions:

  • %: Percentage of parent element's dimension
  • vw/vh: Percentage of viewport width/height
  • em/rem: Relative to font sizes

Content-Based Sizing:

  • fit-content: Shrinks to content size
  • min-content: Minimum size based on content
  • max-content: Ideal size based on content
  • auto: Default - browser calculates based on content and context

Min/Max Constraints: Essential for responsive design:

  • min-width/height: Minimum allowed size
  • max-width/height: Maximum allowed size

Modern CSS Functions:

  • clamp(min, preferred, max): Responsive value within range
  • min(a, b): Smaller of two values
  • max(a, b): Larger of two values

Aspect Ratio: aspect-ratio maintains proportional dimensions (great for responsive images/videos).

Dimensions 
/* Fixed */
width: 300px;
height: 200px;

/* Relative */
width: 50%; /* Parent width */
height: 80vh; /* Viewport height */

/* Minimum & Maximum */
min-width: 300px;
max-width: 1200px;
min-height: 100px;
max-height: 500px;

/* Fill available space */
width: auto; /* default */
width: fit-content; /* shrink to content */
width: max-content; /* max of content */
width: min-content; /* min of content */

/* Modern CSS */
width: clamp(300px, 50%, 800px);
height: min(500px, 80vh);

/* Aspect Ratio */
aspect-ratio: 16 / 9;
aspect-ratio: 1; /* square */

2.5 Display Property

12 min
Intermediate

Chapter Explanation

The display property is one of the most important CSS properties, controlling how an element behaves in the layout and how its children are arranged.

Basic Display Values:

  • block: Takes full width, starts on new line (div, p, headings)
  • inline: Flows with text, only takes needed width (span, a, strong)
  • inline-block: Like inline but accepts width/height and vertical margins
  • none: Completely removed from layout (not just invisible)

Modern Layout Systems:

  • flex: Flexible box layout (one-dimensional)
  • grid: CSS grid layout (two-dimensional)
  • inline-flex / inline-grid: Flex/grid containers that behave like inline elements

Table Layout Values: Mimic HTML table behavior (rarely needed with modern CSS):

  • table, table-row, table-cell

Display vs Visibility:

  • display: none - Element is removed from layout entirely
  • visibility: hidden - Element is invisible but space is preserved

Flow-root: Creates a new block formatting context, useful for containing floats.

Display 
/* Block: full width, new line */
display: block; /* div, p, h1-h6 */

/* Inline: only content width, no new line */
display: inline; /* span, a, strong */

/* Inline-block: inline with block properties */
display: inline-block; /* can have width/height */

/* Flex: flexible box layout */
display: flex;

/* Grid: CSS grid layout */
display: grid;

/* Table */
display: table;
display: table-row;
display: table-cell;

/* None: hide element */
display: none; /* removed from layout */

/* Visibility */
visibility: hidden; /* hidden but space kept */
visibility: visible;

/* Modern */
display: inline-flex;
display: inline-grid;
display: flow-root; /* new block formatting context */

2.6 Positioning

15 min
Intermediate

Chapter Explanation

The position property controls how elements are positioned in the document flow and enables advanced layout techniques.

Position Values:

  • static (default): Normal document flow, ignores top/right/bottom/left
  • relative: Positioned relative to its normal position, stays in document flow
  • absolute: Removed from flow, positioned relative to nearest positioned ancestor
  • fixed: Removed from flow, positioned relative to viewport (stays during scroll)
  • sticky: Hybrid - acts like relative until it crosses a threshold, then like fixed

Offset Properties: Used with positioned elements (not static):

  • top, right, bottom, left
  • Negative values are allowed

Z-index: Controls stacking order (which elements appear on top). Only works on positioned elements. Higher values appear on top.

Positioning Context: Absolute/fixed elements are positioned relative to:

  • Absolute: Nearest ancestor with position ≠ static
  • Fixed: Viewport (or containing block if ancestor has transform/perspective)

Centering Technique: top: 50%; left: 50%; transform: translate(-50%, -50%) centers elements perfectly.

Position 
/* Static: default, normal flow */
position: static;

/* Relative: offset relative to itself */
position: relative;
top: 10px;
left: 20px;

/* Absolute: relative to nearest positioned ancestor */
position: absolute;
top: 0;
right: 0;

/* Fixed: relative to viewport */
position: fixed;
top: 0;
left: 0;

/* Sticky: hybrid of relative & fixed */
position: sticky;
top: 20px;

/* Z-index: stacking order */
z-index: 10; /* higher = on top */
z-index: -1; /* behind content */

/* Position context */
.container {
    position: relative; /* becomes reference for absolute children */
}

.child {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}

Chapter Quiz

  • relative
  • absolute
  • fixed
  • sticky
Answer: fixed or sticky. Fixed always stays at the viewport top, sticky starts relative then becomes fixed after scrolling past a threshold.

2. What must you set on a parent for absolute positioning to work relative to it?

  • position: static;
  • position: relative;
  • display: block;
  • height: 100%;
Answer: position: relative; (or any non-static position). This creates a positioning context for absolute children.

3. What does z-index control?

  • Element size
  • Stacking order (which elements appear on top)
  • Position on the z-axis (3D depth)
  • Zoom level
Answer: Stacking order (which elements appear on top). Higher z-index values appear above lower ones.

2.7 Overflow

Chapter Explanation

The overflow property controls what happens when content overflows its container's bounds. This is crucial for managing content within fixed dimensions.

Overflow Values:

  • visible (default): Content is visible outside container bounds
  • hidden: Content is clipped, no scrollbars
  • scroll: Always shows scrollbars (even if not needed)
  • auto: Shows scrollbars only when needed (most user-friendly)
  • clip: Similar to hidden but prevents programmatic scrolling

Axis-Specific Control:

  • overflow-x: Horizontal overflow only
  • overflow-y: Vertical overflow only

Text Overflow: For single-line text that might overflow:

3.1 Flexbox

18 min
Advanced

Chapter Explanation

Flexbox (Flexible Box Layout) is a one-dimensional layout system designed for distributing space and aligning items in a container. It revolutionized CSS layout when introduced.

Key Concepts:

  • Flex Container: Parent element with display: flex
  • Flex Items: Direct children of flex container
  • Main Axis: Primary axis (horizontal by default)
  • Cross Axis: Perpendicular axis (vertical by default)

Container Properties (control the flex container):

  • flex-direction: Row/column and reverse
  • flex-wrap: Allow items to wrap to new lines
  • justify-content: Alignment along main axis
  • align-items: Alignment along cross axis (single line)
  • align-content: Alignment along cross axis (multiple lines)
  • gap: Space between items (modern, replaces margins)

Item Properties (control flex items):

  • order: Visual reordering (doesn't affect source order)
  • flex-grow: Ability to grow relative to siblings
  • flex-shrink: Ability to shrink relative to siblings
  • flex-basis: Initial size before growing/shrinking
  • flex: Shorthand for grow, shrink, basis
  • align-self: Override container's align-items for specific item

Use Cases: Navigation bars, card layouts, centering content, responsive components.

    flex-direction: row; /* default */

    flex-direction: column-reverse;

    flex-wrap: nowrap; /* default */

    justify-content: center;

    justify-content: space-around;
    justify-content: space-evenly;

    align-items: stretch; /* default */
    align-items: flex-start;
    align-items: flex-end;
    align-items: center;
    align-items: baseline;
    
    /* Align content (multiple lines) */
    align-content: stretch;
    align-content: flex-start;
    align-content: center;
    
    /* Gap */
    gap: 20px;
    row-gap: 10px;

3.2 CSS Grid

CSS Grid is a two-dimensional layout system that allows you to create complex web layouts with rows and columns. It's the most powerful layout system in CSS.

Key Concepts:

  • Grid Container: Parent element with display: grid
  • Grid Items: Direct children of grid container
  • Grid Lines: Dividing lines that make up the grid structure
  • Grid Tracks: Space between two adjacent grid lines (rows or columns)
  • Grid Cells: Single unit of the grid where a row and column intersect
  • Grid Areas: Rectangular space made up of one or more grid cells

Container Properties:

  • grid-template-columns/rows: Define explicit grid tracks
  • grid-template-areas: Name grid areas for visual layout
  • gap: Space between grid tracks
  • justify-items/align-items: Item alignment within cells
  • justify-content/align-content: Grid alignment in container
  • grid-auto-flow: How auto-placed items flow into the grid

Item Properties:

  • grid-column/grid-row: Item placement using line numbers
  • grid-area: Place item in named area or using line numbers
  • justify-self/align-self: Individual item alignment

Grid vs Flexbox: Use Grid for overall page layout (2D), Flexbox for components within grid cells (1D).

Grid Container

    gap: 20px;

    justify-items: stretch; /* default */

    align-content: start;

    grid-auto-rows: 100px;

Responsive Design is the approach of creating websites that work well on all devices and screen sizes. It's not just about mobile devices, but about creating flexible layouts that adapt to any viewing context.

Core Principles:

  • Fluid Grids: Use percentages or fr units instead of fixed pixels
  • Flexible Images: Images that scale within their containers

  • Progressive enhancement (add features for larger screens)
  • Better performance (mobile gets minimal CSS)
  • Forces focus on core content and functionality

Modern Approaches:

  • Container Queries: Style elements based on their container size
  • CSS Clamp: Fluid values with minimum and maximum bounds
  • Intrinsic Web Design: Let content determine breakpoints

3.4 Media Queries

15 min
Advanced

Chapter Explanation

Media Queries are CSS techniques that apply styles based on device characteristics, most commonly screen width. They're the backbone of responsive design, allowing you to create conditional CSS rules.

Media Query Structure:

@media [media-type] [logical-operator] (media-feature) {
  /* CSS rules */
}

Common Media Types:

  • screen (default for web)
  • print (for printing)
  • speech (for screen readers)
  • all (all devices)

Logical Operators:

  • and: Combines multiple conditions
  • , (comma): Acts as OR operator
  • not: Negates the query
  • only: Hides from older browsers

Media Features (Common):

  • width/height: Viewport dimensions
  • min-width/max-width: Minimum/maximum viewport width
  • orientation: landscape or portrait
  • aspect-ratio: Width to height ratio
  • prefers-color-scheme: light or dark mode
  • prefers-reduced-motion: User's motion preference

Breakpoint Strategy:

  • Content-based: Break when content breaks (not device-based)
  • T-shirt sizes: xs, sm, md, lg, xl (with consistent increments)
  • Device-agnostic: 480px, 768px, 1024px, 1200px (common pattern)
Media Queries

/* Multiple conditions */

    .sidebar {
        display: none;
    }
}

/* Orientation */
@media (orientation: landscape) {
    .header {
        height: 100vh;
    }
}

/* Print styles */
@media print {
    .navbar, .footer {
        display: none;
    }
    body {
        font-size: 12pt;
    }
}

/* Dark mode */

    :root {
        --bg: #121212;
        --text: #ffffff;
    }
}

/* Reduced motion */

    * {
        animation-duration: 0.01ms !important;

    /* Desktop styles */
}

@media (min-width: 1441px) {
    /* Large screen styles */
}

CSS Custom Properties (commonly called CSS Variables) allow you to store values in named variables that can be reused throughout your CSS. They're a powerful tool for theming, consistency, and dynamic styling.

Key Characteristics:

  • Scoped: Variables follow CSS cascade and inheritance rules
  • Dynamic: Can be updated with JavaScript at runtime
  • Fallback: Support default values if variable isn't defined

Scoping:

  • Global: Declare in :root or html selector
  • Local: Declare in specific selectors for limited scope

Common Uses:

  • Theming: Dark/light mode, brand color schemes
  • Design Tokens: Centralized color palettes, spacing scales
  • Dynamic Values: Interactive elements, animations
  • Responsive Design: Different values at different breakpoints

JavaScript Integration: Variables can be read and modified with JavaScript:

// Get variable
getComputedStyle(element).getPropertyValue('--variable');

// Set variable
element.style.setProperty('--variable', 'new-value');

Benefits: Consistency, maintainability, theming support, dynamic updates, reduced repetition.

3.7 Transitions & Animations

18 min
Advanced

Chapter Explanation

CSS Transitions and Animations bring interfaces to life by adding motion and visual feedback. They improve user experience by making interactions feel natural and engaging.

CSS Transitions: Simple animations between two states (like hover). They:

  • Animate property changes smoothly
  • Require a trigger (hover, focus, class change)
  • Have four properties: property, duration, timing-function, delay
  • Use hardware acceleration when possible (transform, opacity)

Transition Properties:

  • transition-property: Which properties to animate
  • transition-duration: How long the animation takes
  • transition-timing-function: Pace of animation (ease, linear, etc.)
  • transition-delay: Wait time before starting
  • Shorthand: transition: property duration timing-function delay

CSS Animations: More complex, multi-step animations:

  • Define keyframes with @keyframes
  • Can loop, reverse, pause
  • More control over intermediate steps
  • Don't require a trigger (can run automatically)

Animation Properties:

  • animation-name: Name of the @keyframes rule
  • animation-duration: Length of one cycle
  • animation-timing-function: Pace of animation
  • animation-delay: Start delay
  • animation-iteration-count: How many times to play
  • animation-direction: normal, reverse, alternate, alternate-reverse
  • animation-fill-mode: Styles before/after animation
  • animation-play-state: running or paused

Performance Tips:

  • Animate transform and opacity (hardware accelerated)
  • Avoid animating height, width, margin (causes layout recalculations)
  • Use will-change sparingly to hint browser optimization
  • Respect prefers-reduced-motion for accessibility

Responsive Breakpoint Strategies:

  • Content-Based: Break when content breaks (most effective)
  • Device-Based: Common device sizes (mobile: 320px, tablet: 768px, desktop: 1024px)
  • T-shirt Sizes: xs, sm, md, lg, xl (abstracted from specific pixels)

  • CSS Clamp: Fluid values with min and max bounds
  • Fluid Typography: Text that scales smoothly between sizes
  • Responsive Images: srcset, sizes, picture element
  • Intrinsic Web Design: Let content determine its own layout

Common Layout Patterns:

  • Column Drop: Stack columns vertically on small screens
  • Mostly Fluid: Fluid until breakpoint, then fixed
  • Layout Shifter: Completely rearranges at breakpoints
  • Off Canvas: Navigation hidden off-screen on mobile
  • Grid-Based: CSS Grid with auto-fit and minmax()

Mobile-First Implementation:

  • Start with base (mobile) styles
  • Enhance with min-width media queries
  • Use progressive enhancement
  • Test on real devices (not just emulators)

Performance Considerations:

  • Load images appropriate for device size
  • Conditionally load CSS/JS for different breakpoints
  • Consider touch vs mouse interactions
  • Test connection speeds and data usage

4.5 CSS Performance & Optimization

Chapter Explanation

CSS performance optimization is crucial for fast-loading, responsive websites. Every millisecond counts, especially on mobile devices and slow connections.

CSS Performance Bottlenecks:

  • File Size: Large CSS files increase download time
  • Render Blocking: CSS blocks page rendering
  • Selector Complexity: Complex selectors slow down rendering
  • Expensive Properties: Some CSS properties are performance-heavy
  • Layout Thrashing: Multiple style recalculations

Optimization Strategies:

  • Minification: Remove whitespace, comments, and shorten names
  • Compression: Gzip/Brotli compression for smaller file sizes
  • Critical CSS: Inline above-the-fold styles for immediate rendering
  • Code Splitting: Load only necessary CSS for each page/component
  • Tree Shaking: Remove unused CSS

Rendering Performance:

  • GPU Acceleration: Use transform and opacity for animations
  • Will-change: Hint browser about upcoming changes (use sparingly)
  • Containment: contain property isolates rendering
  • Content-visibility: Skip rendering off-screen content

CSS Delivery Optimization:

  • Preload: <link rel="preload"> for critical CSS
  • Async Loading: Non-critical CSS loaded asynchronously
  • HTTP/2: Multiple simultaneous requests
  • CDN: Distribute CSS files geographically

Build Tools & Processes:

  • PurgeCSS: Remove unused styles
  • PostCSS: Transform CSS with JavaScript plugins
  • CSSNano: Advanced minification
  • Stylelint: Enforce performance best practices
