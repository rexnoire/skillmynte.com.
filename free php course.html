Module 1: PHP Syntax and Tags

PHP (Hypertext Preprocessor) is a widely-used open source server-side scripting language. Unlike HTML, which is static, PHP code is executed on the server, generating dynamic content that is then sent to the client's browser. PHP scripts are embedded into HTML using special tags.

Every PHP statement ends with a semicolon. The basic syntax is <?php // code ?>. You can switch between PHP and HTML modes freely. The echo statement outputs text to the browser, and comments can be added with // or /* */.

Key Points:

  • PHP code is enclosed in <?php ... ?> tags
  • Every statement must end with a semicolon ;
  • Use echo or print to output data
  • Comments: // for single line, /* */ for multi-line
  • Variables start with $ and are dynamically typed
  • You can mix PHP and HTML freely
  • PHP files are processed on the server; clients only see the output

PHP Foundation Explained

PHP is the backbone of countless dynamic websites and web applications, from WordPress to Facebook. Its ability to generate HTML on the fly, interact with databases, and handle user input makes it essential for server-side development. Understanding the basic syntax is the first step toward building interactive, data-driven websites.

The opening and closing tags <?php ?> tell the server to interpret the enclosed code as PHP. Outside these tags, everything is treated as plain HTML. This flexibility allows you to create templates where PHP injects dynamic content into static markup.

Server-Side Processing

PHP runs on the web server, not in the browser. The client receives only the output (usually HTML), so your source code remains hiddenâ€”unlike JavaScript.

Embedding in HTML

You can switch between PHP and HTML as often as needed, making it easy to create dynamic web pages with loops and conditions controlling what HTML is sent.

Database Integration

PHP excels at connecting to databases (MySQL, PostgreSQL, etc.) to retrieve, insert, and manipulate data, forming the core of content management systems and web applications.

Real-World Application

Every time you log into a website, post a comment, or view a personalized dashboard, PHP is likely at work behind the scenes. For example, when you visit an e-commerce site, PHP scripts fetch product details from a database, display them in HTML, and process your shopping cart. Mastering PHP basics opens the door to building such interactive experiences.

Module 2: Variables and Data Types

Variables in PHP are containers for storing data. They always start with a dollar sign ($) followed by the variable name. PHP is loosely typed, meaning you don't have to declare the typeâ€”it is determined by the value assigned.

PHP supports several data types: integers, floats (decimals), strings, booleans, arrays, objects, NULL, and resources. You can check the type of a variable using var_dump() or gettype().

Live Preview (simulated output)

Key Points:

  • All variables start with $ and are case-sensitive
  • No need to declare type; PHP determines it from the value
  • Strings can be enclosed in single or double quotes (double quotes parse variables)
  • Integers are whole numbers; floats are numbers with decimals
  • Booleans represent true or false
  • Arrays hold multiple values indexed numerically or associatively
  • var_dump() displays type and value for debugging

Understanding Data Storage

Variables are fundamental to any programming languageâ€”they act as labeled boxes where you can store, modify, and retrieve data. In PHP, the loose typing system provides flexibility, but you must be aware of type juggling when performing operations between different types.

PHP automatically converts types when necessary (e.g., adding a string and an integer). However, relying on this can lead to bugs; it's often better to cast types explicitly using (int), (string), etc.

Type Juggling

PHP automatically converts types based on context. For example, "5" + 3 results in 8 (integer). This behavior is powerful but can cause unexpected results.

Variable Scope

Variables can have different scopes: global (outside functions), local (inside functions), static (persist across function calls), and superglobals (available everywhere).

Debugging with var_dump

var_dump() is an invaluable tool for inspecting variables during development, showing both the type and value in a readable format.

User Profiles

When building a user profile system, you'll use variables to store data like username (string), age (integer), account balance (float), and whether the user is logged in (boolean). Arrays hold lists of interests or order history. Understanding data types ensures you manipulate this data correctly.

Module 3: Operators and Expressions

Operators are symbols that perform operations on variables and values. PHP includes arithmetic, assignment, comparison, logical, string, and array operators. Expressions are combinations of variables, values, and operators that produce a result.

Understanding operator precedence is crucialâ€”it determines the order in which parts of an expression are evaluated. Parentheses can be used to override precedence.

Try It Yourself:

PHP
Live Preview (simulated output)

Key Points:

  • Arithmetic: + - * / % **
  • Assignment: = += -= *= /= %= .=
  • Comparison: == === != !== < > <= >=
  • Logical: && || !
  • String: . for concatenation
  • Increment/Decrement: ++ --
  • Operator precedence can be controlled with parentheses

Building Expressions

Operators are the tools that let you transform data. Arithmetic operators handle math, comparison operators let you make decisions, logical operators combine conditions, and string operators build text. Mastering operators is essential for writing expressive and efficient code.

A common pitfall is using the assignment operator = instead of the comparison operator == in conditions. Also, note the difference between == (equal value) and === (identical value and type).

Precedence Matters

Just like in math, some operators are evaluated before others. For example, 2 + 3 * 4 equals 14, not 20. Use parentheses to ensure clarity and correct results.

Identical vs Equal

== checks if values are equal after type juggling; === checks both value and type. For robust code, prefer === to avoid surprises.

Concatenation

The dot operator . joins strings. Combined with the assignment operator .=, you can build strings incrementally.

Shopping Cart Calculations

In an e-commerce site, you'll use arithmetic to calculate totals, taxes, and discounts. Comparison operators check if a user is eligible for free shipping. Logical operators validate that a coupon code is both active and not expired. String operators format product names and prices for display.

Module 4: Conditional Statements

Control structures allow your script to make decisions based on conditions. The main conditional statements in PHP are if, else, elseif, and switch. They execute different blocks of code depending on whether a condition evaluates to true or false.

Conditions are typically formed using comparison and logical operators. You can also nest conditionals, but be careful to keep code readable.

Try It Yourself:

PHP
Live Preview (simulated output)

Key Points:

  • if executes code if condition is true; optional else for false
  • elseif checks additional conditions if previous were false
  • switch compares a value against many cases (use break to exit)
  • The ternary operator condition ? value_if_true : value_if_false is a shorthand for simple if-else
  • Conditions can be combined with && (AND), || (OR), ! (NOT)
  • Indentation improves readability

Decision Making

Control structures give your programs intelligence. Without them, code would run the same way every time. By checking conditions, you can create different user experiences based on input, time, user roles, and more.

The switch statement is particularly useful when you have many possible values for a single variable. It can be more readable than a long chain of elseif.

Boolean Context

PHP treats certain values as false in conditions: false, null, 0, 0.0, empty string "", "0", empty arrays. Everything else is true.

Early Returns

In functions, you can return early to avoid deep nesting. This makes code easier to follow and reduces complexity.

Form Validation

Control structures are heavily used in form validation: checking if fields are empty, if email format is valid, if passwords match, etc.

Access Control

In a membership site, you might have code like: if user is logged in, show premium content; elseif user has a trial, show limited content; else show a sign-up page. This logic is implemented with control structures.

LOOPS

Module 5: Loops

Loops allow you to execute a block of code repeatedly. PHP supports while, do-while, for, and foreach loops. Each is suited for different situations: while when you don't know the number of iterations in advance, for when you do, and foreach for iterating over arrays.

Be careful to avoid infinite loops by ensuring the loop condition eventually becomes false.

Try It Yourself:

PHP
Live Preview (simulated output)

Key Points:

  • while checks condition before each iteration
  • do-while checks condition after, so it always runs at least once
  • for is ideal when you know the number of iterations
  • foreach is the easiest way to loop through arrays
  • break exits the loop entirely
  • continue skips to the next iteration
  • Infinite loops happen when the condition never becomes false

Automating Repetition

Loops are essential for handling repetitive tasks efficiently. Instead of writing the same code multiple times, you define the pattern once and let the loop handle the repetitions. This is especially powerful when dealing with dynamic data like database results or user input.

The foreach loop is particularly elegant in PHPâ€”it automatically traverses arrays without needing a counter. When modifying array values inside a loop, remember to use the reference & to affect the original array.

Loop Control

break and continue give you finer control. For example, you can skip invalid data with continue or stop processing after a certain point with break.

Data Processing

Loops are used to iterate over database records, build HTML tables, process file lines, and more. Almost any bulk operation involves a loop.

Performance

Nested loops can become slow with large datasets. Always consider algorithmic efficiency and use PHP's builtâ€‘in array functions when appropriate.

Generating Tables

In a reporting dashboard, you might fetch an array of sales data from the database and use a foreach loop to output each row in an HTML table. Another loop could calculate totals and averages. Loops turn raw data into readable interfaces.

Module 6: Functions

Functions are reusable blocks of code that perform a specific task. They help you avoid repetition, organize your code, and make it more maintainable. You can define your own functions and also use hundreds of builtâ€‘in PHP functions.

A function is declared with the function keyword, can accept parameters, and can return a value. Functions should do one thing well.

  • PHP has thousands of built-in functions for strings, arrays, math, etc.

Building Blocks of Code

Functions are the primary mechanism for abstraction in PHP. By encapsulating logic into named units, you make your code easier to read, test, and reuse. Good function design follows the single responsibility principleâ€”each function should do one thing and do it well.

PHP also supports anonymous functions (closures) and variable functions, which are useful for callbacks and functional programming patterns.

Built-in Power

PHP's standard library includes functions for almost every task: array manipulation, string handling, file operations, database access, and more. Learning them saves time and ensures optimized, tested code.

Reusability

Once you write a function, you can call it from anywhere in your script. This reduces duplication and makes maintenance easierâ€”fix a bug in one place, and it's fixed everywhere.

Scope Isolation

Variables inside a function don't interfere with variables outside, preventing accidental side effects. This isolation makes code more predictable.

Utility Libraries

In a typical web application, you'll have utility functions like formatDate(), sanitizeInput(), or calculateDiscount(). These functions are used throughout the app, ensuring consistency and centralizing logic.

Module 7: Arrays

Arrays are variables that can hold multiple values at once. PHP arrays are actually ordered mapsâ€”they can be indexed numerically (indexed arrays) or by strings (associative arrays). They are incredibly flexible and are used everywhere in PHP.

PHP provides a huge set of array functions for sorting, filtering, merging, searching, and more. Learning these functions will save you countless hours.

Managing Collections

Arrays are the workhorses of PHP. They're used to store lists, map relationships, hold database records, manage form data, and much more. Mastering array functions is key to writing concise and efficient PHP code.

Unlike some languages, PHP arrays are ordered mapsâ€”they preserve insertion order and can have mixed keys. This flexibility makes them suitable for a wide range of data structures.

Filtering and Mapping

Functions like array_filter(), array_map(), and array_reduce() let you process arrays functionally, reducing the need for explicit loops.

Searching

in_array(), array_search(), and array_key_exists() help you find values or keys quickly.

Multidimensional Arrays

Arrays of arrays are commonâ€”for example, a table of database rows. You can access nested data with multiple indices (e.g., $data[0]['name']).

Shopping Cart

A shopping cart is often stored as an array of items, each item being an associative array with keys like "id", "name", "price", "quantity". Array functions calculate total price, apply discounts, and sort items.

Module 8: Superglobals and Form Handling

PHP provides several superglobal arrays that are accessible from anywhere in your script. They contain information about the server, request, session, etc. The most important for form handling are $_GET and $_POST, which hold data sent via HTTP GET or POST methods.

When a user submits a form, PHP automatically populates these superglobals. You can then access the data, validate it, sanitize it, and use it (e.g., store in a database, send an email). Always validate and sanitize user input for security.

Try It Yourself:

PHP
Live Preview (simulated output)

Key Points:

  • $_GET contains data from URL query string (method="get")
  • $_POST contains data from HTTP POST (method="post")
  • $_REQUEST combines GET, POST, and COOKIE
  • $_SERVER holds server and request information
  • Always validate and sanitize input: use filter_var(), htmlspecialchars(), etc.
  • Check if form was submitted using $_SERVER["REQUEST_METHOD"]
  • Use null coalescing operator ?? to avoid undefined index notices

Interacting with Users

Forms are the primary way users interact with web applicationsâ€”logging in, registering, submitting comments, making purchases. PHP's superglobals make it easy to retrieve this data, but security is paramount. Never trust user input; always validate and sanitize.

Crossâ€‘Site Scripting (XSS) is a common vulnerability where attackers inject malicious code. Using htmlspecialchars() when outputting user data prevents this. For database interactions, use prepared statements to prevent SQL injection.

Validation vs Sanitization

Validation checks if the data meets criteria (e.g., email format). Sanitization cleans the data (e.g., removing tags). Both are essential.

Other Superglobals

$_SESSION stores user-specific data across pages, $_COOKIE holds client-side data, $_FILES handles file uploads, and $_ENV contains environment variables.

Self-Submitting Forms

Setting the form action="" submits to the same page, allowing you to process and display the form in one script. This is common in MVC patterns.

Contact Forms

Almost every website has a contact form. The user fills it out, PHP validates the input, and if valid, sends an email using mail() or a library like PHPMailer. Error messages are displayed if validation fails.

FILE HANDLING

Intermediate

Module 9: Working with Files

PHP can create, read, write, and delete files on the server. This is useful for storing data, creating logs, handling file uploads, and more. The main functions are fopen(), fread(), fwrite(), fclose(), and simpler alternatives like file_get_contents() and file_put_contents().

Always check that you have proper permissions and that the file operations succeed. Use error handling to avoid unexpected script termination.

Try It Yourself:

PHP
Live Preview (simulated output)

Key Points:

  • file_get_contents() reads entire file into string
  • file_put_contents() writes string to file (overwrite by default, use FILE_APPEND to add)
  • fopen() opens a file with a mode: r (read), w (write), a (append), etc.
  • Always fclose() after using fopen()
  • Check file_exists() before reading or writing
  • filesize() returns size in bytes
  • unlink() deletes a file

Persistent Data

File handling allows your application to store data beyond a single request. While databases are often preferred for structured data, files are perfect for configuration, logs, caching, and simple data storage.

Security is crucial when dealing with file operations. Never allow user input to determine file paths directly, as this can lead to directory traversal attacks. Validate and sanitize any userâ€‘supplied filenames.

Permissions

Your PHP script must have appropriate permissions to read/write files. On shared hosting, set correct file and directory permissions (e.g., 644 for files, 755 for directories).

Logging

Applications often log errors, access, or custom events to files. Functions like error_log() can write to a log file for later analysis.

File Uploads

PHP can handle file uploads from forms via $_FILES. The uploaded file is temporarily stored, then you move it to a permanent location with move_uploaded_file().

Configuration Files

Many PHP applications store settings in .ini or .php files. For example, a CMS might have a config.php file with database credentials. Using include or parse_ini_file(), you load these settings into your script.
