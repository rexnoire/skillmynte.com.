Module 2: Basic Routing & Route Parameters

Routes are the entry points to your Laravel application. They map HTTP requests (like GET or POST) to specific controllers or closures. Laravel's routing is clean and expressive, supporting route parameters, named routes, and groups.

All routes are defined in files inside the routes directory. The most common is web.php for browserâ€‘facing routes. You can also define API routes in api.php.

Key Points:

  • Routes are defined using Route:: methods (get, post, put, delete, etc.)
  • Route parameters are enclosed in curly braces { }
  • Optional parameters have a trailing ? and must have a default value
  • Named routes allow easy URL generation via route('name')
  • Route groups can share attributes like middleware, prefixes, and namespaces
  • You can bind models directly to routes using routeâ€‘model binding

How Routing Works

When a request hits your Laravel application, the framework parses the URL and matches it against your defined routes. The first matching route determines which closure or controller method will handle the request. This system allows you to build clean, RESTful URLs that map directly to application logic.

HTTP Verbs

Laravel supports all standard HTTP verbs: GET, POST, PUT, PATCH, DELETE, and OPTIONS. This makes it easy to build RESTful APIs.

Named Routes

Named routes let you generate URLs without hardâ€‘coding paths. For example, route('profile') returns the URL for the profile route, even if you later change the path.

Middleware

Routes can be assigned middleware that runs before or after the request. This is commonly used for authentication, logging, or CORS handling.

Eâ€‘commerce Example

An online store might have routes like: /products (list), /products/{id} (details), /cart/add/{id} (POST), and /checkout (GET/POST). Each route would be mapped to a controller method, keeping the application organized and scalable.

Organize request handling logic

Module 3: Controllers & Resource Controllers

Controllers group related request handling logic into a single class. Instead of defining all your route logic in closures, you can move it to controller methods. This keeps your routes file clean and promotes code reusability.

Laravel also provides resource controllers that automatically handle typical CRUD operations (index, create, store, show, edit, update, destroy) with a single line in your routes file.

Key Points:

  • Controllers are stored in app/Http/Controllers
  • Use php artisan make:controller NameController to generate a controller
  • Resource controllers provide methods for CRUD operations
  • Single action controllers can be created with --invokable flag
  • Controllers can return views, redirects, JSON responses, etc.
  • Middleware can be assigned to controllers in the constructor

Why Controllers?

Controllers centralize your application's business logic. Instead of scattering logic across many closures, each controller handles all requests related to a specific resource. This makes your code easier to test, maintain, and extend. Laravel's dependency injection also works seamlessly in controller methods.

Resourceful Routing

Using Route::resource() automatically creates the seven RESTful routes and maps them to the corresponding controller methods. It's a huge timeâ€‘saver.

Single Action Controllers

If a controller only needs one action (like a dashboard), you can make it invokable by defining an __invoke() method and using Route::get('/dashboard', DashboardController::class).

Dependency Injection

Controller methods can typeâ€‘hint any dependencies (like Request, Model, or custom services), and Laravel's service container will automatically resolve them.

Blog Example

For a blog, you'd likely have a PostController with methods: index() (list posts), show() (display one post), create()/store() (new post), edit()/update() (modify post), and destroy() (delete). Using a resource controller, a single line Route::resource('posts', PostController::class) sets up all these routes.

BLADE TEMPLATING

Create dynamic views with Blade

Intermediate

Module 4: Blade Syntax & Layouts

Blade is Laravel's powerful templating engine. It allows you to write plain PHP code in your views while providing convenient shortcuts for common tasks like displaying data, control structures, and template inheritance. Blade views are compiled into plain PHP and cached until modified, so they add zero overhead to your application.

Blade uses {{ }} to escape output and {!! !!} for unescaped output. It also provides directives like @if, @foreach, @include, and @extends for layout inheritance.

Key Points:

  • Blade files use the .blade.php extension
  • {{ $variable }} automatically escapes output (prevents XSS)
  • @if, @elseif, @else, @endif for conditionals
  • @foreach, @forelse, @for, @while for loops
  • @include('view.name') includes a subâ€‘view
  • @extends('layout') and @section/@yield enable template inheritance
  • Blade also has components, slots, and stacks for advanced features

Why Blade?

Blade combines the simplicity of plain PHP with convenient shortcuts. It doesn't restrict you from using raw PHP when needed, but its directives make common tasks (like loops and conditionals) much cleaner. Template inheritance eliminates repetitive HTML, and components let you build reusable UI pieces.

Automatic Escaping

By default, {{ }} escapes output, protecting your application from XSS attacks. When you need to output raw HTML (e.g., from a WYSIWYG editor), use {!! !!} with caution.

Components & Slots

Laravel's Blade components allow you to create reusable HTML elements with their own logic. They can be classâ€‘based or anonymous, and support slots for flexible content insertion.

Stacks

Stacks let you push content from child views to specific places in a layout (e.g., adding scripts to the bottom of the page). This is great for organizing assets.

CMS Example

A content management system built with Laravel would use Blade extensively. The admin panel might have a layout with a sidebar and top bar; each content section (pages, posts, media) extends that layout and injects its own content. Components would be used for forms, buttons, and data tables, keeping the code DRY and maintainable.

Module 5: Models, Migrations & Querying

Eloquent is Laravel's Objectâ€‘Relational Mapper (ORM) that provides an ActiveRecord implementation to work with your database. Each database table has a corresponding Model that you use to interact with that table. Eloquent makes common database operations intuitive and expressive.

Migrations are version control for your database. They allow you to define and share your application's database schema. You can roll back changes and keep your team's databases in sync.

The Power of Eloquent

Eloquent turns complex SQL queries into chainable, readable PHP. It also provides events, accessors/mutators, and scopes to encapsulate common query logic. Because each model is a fullâ€‘fledged PHP class, you can add your own methods to extend functionality.

Timestamps

By default, Eloquent expects created_at and updated_at columns and manages them automatically.

Query Scopes

Local scopes allow you to define common query constraints in the model, e.g., scopePopular($query) to filter popular posts.

Accessors & Mutators

You can format attribute values when retrieving or setting them. For example, automatically hash a password when setting it.

Analytics Dashboard

An analytics dashboard might query millions of records. Eloquent's whereBetween, groupBy, and aggregate methods (count, sum, avg) make it easy to build reports. With proper indexing and query optimization, Eloquent can handle large datasets efficiently.

Module 6: Form Handling & Validation

Laravel makes it easy to handle form submissions and validate incoming data. With builtâ€‘in CSRF protection, validation rules, and error handling, you can build secure forms quickly. Blade's @csrf directive adds a hidden CSRF token, and the old() helper repopulates forms after validation errors.

Sometimes you need to perform additional checks after validation (e.g., verify a user's subscription). Form requests have an after() method for this purpose.

Registration Form

A typical registration form validates that the name, email, and password are present, the email is unique, and the password is confirmed and meets strength requirements. Laravel's validation makes this trivial, and the builtâ€‘in error handling provides a smooth user experience.
