What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a web page and consists of a series of elements that tell the browser how to display content.

Key Concepts:

• HTML is not a programming language - it's a markup language

• HTML uses tags to define elements

• HTML documents are saved with .html extension

• Every web page you visit is built with HTML

HTML 
<!DOCTYPE html>
<html>
<head>
    <title>My First Web Page</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

HTML Document Structure

Every HTML document follows a specific structure. Understanding this structure is crucial for creating valid web pages.

Basic Structure:

1. DOCTYPE Declaration - Defines the HTML version

2. html element - Root element of the page

3. head section - Contains metadata and links

4. body section - Contains visible content

HTML 
<!DOCTYPE html>
<!-- This tells browsers we're using HTML5 -->

<html lang="en">
<!-- lang attribute helps with accessibility and SEO -->

<head>
    <!-- Meta information about the page -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>

<body>
    <!-- Visible content goes here -->
    <h1>Main Heading</h1>
</body>

</html>

Headings & Paragraphs

Headings and paragraphs are the most fundamental content elements in HTML. They provide structure and readability to your content.

Heading Hierarchy:

HTML provides six levels of headings, from h1 (most important) to h6 (least important).

HTML

<h3>Subsection 1.1</h3>
<p>Detailed content...</p>

<h2>Section 2</h2>

Paragraph Formatting:

Paragraphs automatically create space above and below. You can use line breaks within paragraphs when needed.

HTML 
<!-- Multiple paragraphs -->
<p>This is the first paragraph. It contains multiple sentences to form a cohesive thought.</p>

<p>This is the second paragraph. Notice the automatic spacing between paragraphs.</p>

<!-- Line breaks within a paragraph -->
<p>This is the first line.<br>
This is the second line in the same paragraph.<br>
This is the third line.</p>

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the opening tag and usually come in name/value pairs.

Common Attributes:

id: Unique identifier for an element

class: Class name for styling and JavaScript

src: Source for images, scripts, etc.

href: Hyperlink reference

alt: Alternative text for images

style: Inline CSS styles

HTML 
<!-- Element with multiple attributes -->

   target="_blank"
   title="Visit Example Website">

Some attributes don't need a value - their presence alone indicates true.

HTML Comments

Comments are used to leave notes in HTML code that are not displayed in the browser. They are essential for documentation and code organization.

Comment Syntax:

HTML comments start with <!-- and end with -->

HTML 
<!-- This is a single-line comment -->

<!-- 
    This is a multi-line comment.
    It can span multiple lines.
    Useful for longer explanations.
-->

<div class="container">
    <!-- Navigation Section -->
    <nav>
        <!-- TODO: Add more navigation items -->
        <a href="#">Home</a>
    </nav>
    
    <!-- Main Content Area -->
    <main>
        <h1>Welcome</h1>
        <!-- Last updated: 2024 -->
    </main>
</div>

Best Practices for Comments:

1. Document sections: Use comments to separate different parts of your HTML

2. TODO comments: Mark areas that need work

3. Explain complex code: Comment on why something is done a certain way

4. Temporary removal: Comment out code instead of deleting it

HTML 
<!-- ============================================
     HEADER SECTION
     Contains logo and main navigation
============================================ -->
<header id="main-header">
    <div class="logo">
        <img src="logo.png" alt="Company Logo">
    </div>
    
    <!-- TODO: Add mobile menu toggle -->
    <nav class="main-nav">
        <!-- Navigation items go here -->
    </nav>
</header>

<!-- ============================================
     OLD CODE - KEPT FOR REFERENCE
     <div class="old-banner">
         <p>Old content here</p>
     </div>
============================================ -->

Links (Anchor Elements)

The anchor element (<a>) creates hyperlinks to other web pages, files, locations within the same page, or email addresses.

Basic Link Structure:

HTML 
<!-- External link -->
<a href="https://www.example.com">
    Visit Example Website
</a>

<!-- Internal link -->
<a href="/about.html">
    About Us
</a>

<!-- Link with target -->
<a href="https://www.example.com" 
   target="_blank"
   rel="noopener noreferrer">
    Open in New Tab
</a>

<!-- Link with title tooltip -->
<a href="#section2" 
   title="Jump to Section 2">
    Go to Section 2
</a>

Special Link Types:

HTML 
<!-- Email link -->
<a href="mailto:contact@example.com">
    Email Us
</a>

<a href="mailto:contact@example.com?subject=Inquiry&body=Hello,">
    Email with Subject
</a>

<!-- Phone link -->
<a href="tel:+1234567890">
    Call Us: +1 (234) 567-890
</a>

<!-- Download link -->
<a href="/documents/report.pdf" download>
    Download Report (PDF)
</a>

<a href="/documents/report.pdf" 
   download="annual-report-2024.pdf">
    Download with Custom Filename
</a>

Page Anchors (Internal Navigation):

HTML 
<!-- Navigation menu with anchors -->
<nav>
    <a href="#home">Home</a>
    <a href="#about">About</a>
    <a href="#services">Services</a>
    <a href="#contact">Contact</a>

    <h2>About Section</h2>

Images & Alt Text

Basic Image Syntax:

HTML

     height="100">

<img src="banner.jpg" 
     alt="Website banner showing our services"
     style="max-width: 100%; height: auto;">

Image with Figure and Caption:

HTML 
<!-- Image with figure and caption -->
<figure>
    <img src="graph.jpg" 
         alt="Bar chart showing sales growth from 2020 to 2024"
         width="600"
         height="400">
    <figcaption>
        Figure 1: Quarterly sales growth from 2020-2024
    </figcaption>

<figure class="gallery">
    <img src="product1.jpg" 
         alt="Wireless headphones in black color">

    <img src="product3.jpg" 
         alt="Wireless headphones with charging case">

Responsive Images with srcset:

HTML 
<!-- Responsive image with srcset -->

Audio & Video

HTML provides native support for embedding audio and video content without requiring plugins like Flash.

Audio Element:

HTML 
<!-- Basic audio player -->
<audio controls>
    <source src="audio/song.mp3" type="audio/mpeg">
    <source src="audio/song.ogg" type="audio/ogg">
    <p>Your browser doesn't support HTML5 audio. 
        <a href="audio/song.mp3">Download the audio file instead.</a>
    </p>
</audio>

<!-- Audio with more options -->
<audio controls 
       autoplay 
       loop
       muted
       preload="auto">
    <source src="background-music.mp3" type="audio/mpeg">
</audio>

Video Element:

HTML 
<!-- Basic video player -->
<video controls width="640" height="360">
    <source src="video/movie.mp4" type="video/mp4">
    <source src="video/movie.webm" type="video/webm">
    <track src="subtitles-en.vtt" 
           kind="subtitles" 
           srclang="en" 
           label="English" 
           default>
    <p>Your browser doesn't support HTML5 video. 
        <a href="video/movie.mp4">Download the video instead.</a>
    </p>
</video>

<!-- Responsive video container -->
<div class="video-container">
    <video controls 
           playsinline
           poster="video-thumbnail.jpg">
        <source src="video/tutorial.mp4" type="video/mp4">
    </video>
</div>

Audio and Video Attributes:

controls: Shows playback controls (play, pause, volume)

autoplay: Starts playing automatically (browsers may block this)

loop: Replays media when it ends

muted: Starts with audio muted

preload: "auto", "metadata", or "none" for loading behavior

poster: Image shown before video plays

playsinline: For mobile - plays video inline instead of fullscreen

HTML

        height="315"

        allowfullscreen>
</iframe>

<!-- Audio with fallback for very old browsers -->
<audio controls>
    <source src="audio/podcast.mp3" type="audio/mpeg">
    <source src="audio/podcast.oga" type="audio/ogg">
    <!-- Fallback for IE8 -->
    <embed src="audio/podcast.mp3" 
           type="audio/mpeg" 
           width="300" 
           height="30">

Forms

HTML forms collect user input and send it to a server for processing. Forms contain various input elements and controls.

Basic Form Structure:

HTML 
<!-- Simple contact form -->
<form action="/submit-form" method="POST">
    <div>
        <label for="name">Full Name:</label>
        <input type="text" id="name" name="name" required>
    </div>
    
    <div>
        <label for="email">Email Address:</label>
        <input type="email" id="email" name="email" required>
    </div>
    
    <div>
        <label for="message">Message:</label>
        <textarea id="message" name="message" rows="5" required></textarea>
    </div>
    
    <button type="submit">Send Message</button>
</form>

Form Attributes:

action: URL where form data is sent

method: "GET" (data in URL) or "POST" (data in request body)

enctype: Encoding type for file uploads (multipart/form-data)

target: Where to open response (_self, _blank, etc.)

novalidate: Disables browser validation

autocomplete: "on" or "off" for form autofill

HTML 
<!-- Form with various attributes -->
<form action="/upload"
      method="POST"
      enctype="multipart/form-data"
      target="_blank"
      novalidate
      autocomplete="off"
      id="upload-form">
    
    <fieldset>

    const password = document.getElementById('user-password');

        document.getElementById('password-error').textContent = '';
    }
    
    // Confirm password validation
    if (password.value !== confirmPassword.value) {
        document.getElementById('confirm-error').textContent = 'Passwords do not match';
        isValid = false;
    } else {
        document.getElementById('confirm-error').textContent = '';
    }
    
    if (isValid) {
        alert('Registration successful!');
        this.reset();
    }
});

    <p>Paragraph <div>Nested div - INVALID!</div></p>

    <!-- Duplicate IDs -->
    <div id="main">Content</div>
    <div id="main">More content - INVALID!</div>
    
    <!-- Deprecated elements -->
    <center>This is deprecated</center>
    <font size="3">Deprecated font tag</font>

<div class="container">
    <!-- Proper image with alt text -->
    <img src="image.jpg" alt="Descriptive text about the image">
    
    <!-- Proper nesting -->
    <div>
        <p>Paragraph with <span>inline element</span> inside</p>
    </div>
    
    <!-- All tags properly closed -->
    <div class="wrapper">
        <p>Properly closed paragraph</p>
    </div>
    
    <!-- Unique IDs -->
    <div id="main-content">Content</div>
    <div id="sidebar">More content</div>
    
    <!-- Modern CSS instead of deprecated elements -->
